        - --- 補足：用語の説明 ----
        
        Server Component とは：
        サーバー側で実行されるコンポーネント。
        Next.js の App Router では、デフォルトで全てのコンポーネントが
        Server Component として扱われる。
        ファイルの先頭に "use client" の記述がなければ Server Component。
        
        Client Component とは：
        ブラウザ側で実行されるコンポーネント。
        ファイルの先頭に "use client" と記述されているものが Client Component。
        ボタンのクリックや入力欄の操作など、ユーザーの操作に反応する
        処理が必要な場合に使う。
        
        Server Component と Client Component の見分け方：
        
        ```
        ファイルの先頭を見る。
        
          "use client" の記述あり → Client Component
          "use client" の記述なし → Server Component
        ```
        
        await params とは：
        Server Component で params（URL のパラメータ）を取得する方法。
        Server Component は async function（非同期関数）として
        定義できるため、await を使って params を取得する。
        
        ```
          例：
            export default async function Page({ params }) {
              const { slug } = await params
            }
        ```
        
        use(params) とは：
        Client Component で params を取得する方法。
        Client Component は async function として定義できないため、
        React が提供する use という関数を使って params を取得する。
        
        ```
          例：
            "use client"
            import { use } from "react"
        
            export default function Page({ params }) {
              const { slug } = use(params)
            }
        ```
