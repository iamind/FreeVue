## Next.js 14.2.32 → v16 変更点一覧（概要説明付き）

### 1. ランタイム・依存関係の要件変更

**概要：** Next.jsを動かすために必要なソフトウェアの最低バージョンが上がった。

| 用語 | 説明 |
|------|------|
| Node.js | JavaScriptをサーバー側で実行するための環境。Next.jsはNode.js上で動作する |
| React / React DOM | UIを構築するためのライブラリ。Next.jsはReactをベースにしている |
| TypeScript | JavaScriptに型を追加した言語。型チェックでバグを事前に防げる |

---

### 2. Async Request APIs（破壊的変更・最重要）

**概要：** サーバー側でリクエスト情報（Cookie、ヘッダー、URLパラメータなど）を取得するための関数の使い方が変わった。

| 用語 | 説明 |
|------|------|
| `cookies()` | ブラウザから送られてきたCookie（ログイン情報など）を取得する関数 |
| `headers()` | リクエストのHTTPヘッダー（User-Agent、認証トークンなど）を取得する関数 |
| `draftMode()` | CMSの下書きプレビュー機能を制御する関数 |
| `params` | URLの動的部分（例：`/blog/[slug]`の`slug`）を取得するオブジェクト |
| `searchParams` | URLのクエリパラメータ（例：`?page=2`の`page`）を取得するオブジェクト |
| 非同期（async/await） | 処理の完了を待ってから次に進む仕組み。`await`をつけて結果を待つ必要がある |

---

### 3. キャッシュ動作の変更（破壊的変更）

**概要：** データの一時保存（キャッシュ）の初期設定が「保存する」から「保存しない」に変わった。

| 用語 | 説明 |
|------|------|
| キャッシュ | 一度取得したデータを一時的に保存し、次回は保存したデータを返すことで高速化する仕組み |
| `fetch` | 外部APIやサーバーからデータを取得するための関数 |
| Route Handlers | `app/api/`配下に作成するAPIエンドポイント。サーバー側で処理を行う |
| Client Router Cache | ページ遷移時にブラウザ側でページデータを保持し、再利用する仕組み |

---

### 4. Turbopackがデフォルトに

**概要：** 開発・ビルド時にコードをまとめる処理（バンドル）を行うツールが、WebpackからTurbopackに変わった。

| 用語 | 説明 |
|------|------|
| Turbopack | Vercel社が開発した新しいバンドラー。Webpackより高速 |
| Webpack | 従来のバンドラー。多くのプロジェクトで長年使われてきた |
| バンドラー | 複数のJavaScript/CSSファイルを1つにまとめ、ブラウザで効率的に読み込めるようにするツール |

---

### 5. React 19関連の変更

**概要：** React本体のバージョンアップに伴い、一部のフック（状態管理の関数）の名前や機能が変わった。

| 用語 | 説明 |
|------|------|
| `useFormState` | フォーム送信の状態を管理するフック。v19で非推奨になった |
| `useActionState` | `useFormState`の後継。送信中かどうか（pending）も直接取得できる |
| `useFormStatus` | フォームの送信状態（送信中かどうか等）を取得するフック |
| フック（Hook） | Reactコンポーネント内で状態や副作用を扱うための関数（`use`で始まる） |

---

### 6. next.config.js の設定変更

**概要：** Next.jsの設定ファイル内のオプション名が変更・移動された。

| 用語 | 説明 |
|------|------|
| next.config.js | Next.jsの動作をカスタマイズするための設定ファイル |
| `experimental` | 実験的機能を有効にするためのオプション群。安定化すると外に移動する |
| `cacheComponents` | コンポーネント単位でキャッシュを制御する新機能を有効にするオプション |

---

### 7. 削除された機能

**概要：** 以前は使えていた機能やパッケージが完全に削除され、使用するとエラーになる。

| 用語 | 説明 |
|------|------|
| `next lint` | Next.js組み込みのコード品質チェックコマンド。削除された |
| `serverRuntimeConfig` / `publicRuntimeConfig` | 実行時に設定値を読み込む旧方式。環境変数に置き換え |
| AMP | Googleが推進していたモバイル高速化技術。採用が減少したため削除 |
| `@next/font` | フォント最適化の旧パッケージ。`next/font`に統合された |

---

### 8. ESLint関連

**概要：** コードの品質・スタイルをチェックするツール（ESLint）の使い方が変わった。

| 用語 | 説明 |
|------|------|
| ESLint | JavaScriptのコード品質をチェックするツール。バグや書き方の問題を検出する |
| Flat Config | ESLintの新しい設定ファイル形式。従来の`.eslintrc`形式から移行が推奨される |
| `next build` | 本番用にアプリをビルドするコマンド。以前はリントも実行していたが、v16では実行しない |

---

### 9. Parallel Routes

**概要：** 1つのページ内に複数の独立したルート（画面）を同時に表示する機能に関する要件が厳しくなった。

| 用語 | 説明 |
|------|------|
| Parallel Routes | `@modal`のようなスロットを使い、モーダルやサイドバーを独立したルートとして扱う機能 |
| `default.js` | Parallel Routeのスロットが表示されないときに代わりに表示するファイル。v16で必須になった |

---

### 10. 開発環境の変更

**概要：** 開発時とビルド時の出力先が分離され、同時に実行できるようになった。

| 用語 | 説明 |
|------|------|
| `.next/` | Next.jsがビルド結果を出力するフォルダ |
| `.next/dev` | v16で追加された開発専用の出力先 |

---

### 11. その他の変更

| 用語 | 説明 |
|------|------|
| `devIndicators` | 開発中に画面に表示されるインジケーター（ISR状態など）の設定 |
| `scroll-behavior: smooth` | ページ内スクロールを滑らかにするCSSプロパティ |
| Sassのチルダインポート | `@import '~bootstrap/...'`のような書き方。Turbopackでは非サポート |
